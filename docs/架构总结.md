## 1. 项目概览

TenetCpp 是一个高性能张量网络量子模拟框架，尽管名字含有"Cpp"，实际完全用 Julia 实现。框架围绕矩阵乘积态（MPS）和矩阵乘积算符（MPO）构建，深度集成对称性保持的张量运算，支持多种量子多体算法。

**主要特点：**
- 支持 SU(2)、U(1) 及其乘积等量子数守恒的对称张量
- 同时提供稠密（Dense）和稀疏（Sparse）两种表示
- 实现了 DMRG、TDVP、CBE、SETTN 等主流算法
- 基于扫描（Sweep）框架，支持单格点和双格点更新
- 内置可观测量测量树，支持任意多体算符的期望值计算

## 3. 模块详解

### 3.1 核心配置

#### `Globals.jl`

关闭 TensorKit 的 braid 缓存以优化性能：

```julia
TensorKit.usebraidcache_abelian[] = false
TensorKit.usebraidcache_nonabelian[] = false

get_num_cpus() = Sys.CPU_THREADS
get_num_threads_julia() = Threads.nthreads()
```

#### `Defaults.jl`

预定义两个 Lanczos 求解器参数：

```julia
# DMRG 用（精度要求低，速度优先）
global DMRGDefaultLanczos = Krylovalgo(KrylovKit.Lanczos(
    krylovdim = 8,
    maxiter   = 1,
    tol       = 1e-6,
    orth      = ModifiedGramSchmidt(),
    eager     = true,
    verbosity = 0
))

# TDVP 用（精度要求高）
global TDVPDefaultLanczos = Krylovalgo(KrylovKit.Lanczos(
    krylovdim = 32,
    maxiter   = 1,
    tol       = 1e-8,
    orth      = ModifiedGramSchmidt(),
    eager     = true,
    verbosity = 0
))
```

#### `init.jl`

运行时初始化：配置 Julia 多线程、BLAS 线程数，检测 MKL 并打印诊断信息。

struct DenseMPS{L, T}

### 3.5 环境张量 (Environment)

环境张量存储左右部分缩并结果，是扫描算法的核心缓存结构。

#### 稠密环境张量

| 类型 | 说明 |
|------|------|
| `LocalEnvironmentTensor{R}` | 单格点环境 |
| `LeftEnvironmentTensor{R}` | 左侧部分缩并 |
| `RightEnvironmentTensor{R}` | 右侧部分缩并 |
| `LeftCompositeEnvironmentTensor{Rcd,Rt,N,I}` | 复合左环境 |
| `RightCompositeEnvironmentTensor{Rcd,Rt,N,I}` | 复合右环境 |
| `DenseLeftEnvironmentTensor{R}` | 稠密左环境 |
| `DenseRightEnvironmentTensor{R}` | 稠密右环境 |

#### 稀疏环境张量

| 类型 | 说明 |
|------|------|
| `SparseEnvironmentTensor` | 稀疏环境（含向量 + 键维 D） |
| `SparseLeftEnvironmentTensor{N}` | N 层稀疏左环境 |
| `SparseRightEnvironmentTensor{N}` | N 层稀疏右环境 |

#### 主环境结构

**`Environment{N, L}`**（N = 层数，L = 格点数）

```julia
struct Environment{N, L}
    layer  :: Vector             # N 层：[MPS, MPO, adjoint(MPS)]
    envs   :: Union{Nothing, Array{AbstractEnvironmentTensor}}  # 缓存张量
    center :: Vector{Int64}      # 当前中心位置
    L      :: Int64              # 系统大小
end
```

典型三层构建：
```julia
env = Environment([ψ, H, adjoint(ψ)])
```

#### CBE 专用环境

**`CBEenvironment`**

```julia
struct CBEenvironment
    tL₀, tR₀  :: AbstractTensorWrapper        # 参考左/右张量
    tL, tR     :: Union{AbstractTensorWrapper, Nothing}  # 工作张量
    D_i, D_f   :: Int64                        # 初始/最终键维度
    Λ          :: Union{AbstractTensorWrapper, Nothing}  # 奇异值
    Lorth, Rorth :: Union{SparseLeftEnvironmentTensor, ...}
end
```

### 3.6 哈密顿量 (Hamiltonian)

#### 稀疏投影哈密顿量

**`SparseProjectiveHamiltonian{N}`**

```julia
struct SparseProjectiveHamiltonian{N}
    EnvL      :: SparseLeftEnvironmentTensor   # 左环境
    EnvR      :: SparseRightEnvironmentTensor  # 右环境
    H         :: Union{Nothing, SparseMPO}     # MPO（N≥1 时非空）
    validinds :: Tuple                         # 有效索引组合

end
```

**构建函数：**

```julia
proj0(EnvL, EnvR; E₀)          # 0 格点（恒等投影）
proj1(env, site; E₀)            # 单格点有效哈密顿量
proj2(env, site1, site2; E₀)    # 双格点有效哈密顿量
projleft0(env; E₀)              # 最左边界投影
projright0(env; E₀)             # 最右边界投影
```

#### 稠密投影哈密顿量

**`DenseProjectiveHamiltonian{N, L}`**

用于需要显式矩阵表示的场景。

### 3.7 相互作用树 (IntrTree)

相互作用树是构建 `SparseMPO` 的核心数据结构，用于将用户定义的 Hamiltonian 转换为 MPO 形式。

#### 局域算符 (`LocalOperator.jl`)

**`InteractionTreeLeave{N}`**（树叶，存储一条相互作用项）

```julia
struct InteractionTreeLeave{N}
    A        :: NTuple{N, AbstractTensorMap}  # N 个算符张量
    site     :: NTuple{N, Int64}              # 格点索引（有序）
    name     :: NTuple{N, String}             # 算符名称
    fermionic:: NTuple{N, Bool}               # Jordan-Wigner 弦标记
    strength :: Number                        # 耦合系数
    Z        :: Union{Nothing, AbstractTensorMap}  # 辅助张量
end
```

**`LocalOperator{R₁, R₂}`**（单格点算符）

```julia
struct LocalOperator{R₁, R₂} <: AbstractLocalOperator{R₁, R₂}
    A        :: Union{Nothing, AbstractTensorMap}
    name     :: String
    site     :: Int64
    strength :: Union{Nothing, Number}
end
```

**`IdentityOperator{R}`**（恒等算符）

```julia
struct IdentityOperator{R} <: AbstractLocalOperator{0, 0}
    A        :: Union{Nothing, AbstractTensorMap}
    site     :: Int64
    strength :: Number
    name     :: Union{Nothing, String, Tuple}
end
```

#### 树节点 (`Node.jl`)

**`InteractionTreeNode`**

```julia
mutable struct InteractionTreeNode <: AbstractTreeNode
    A        :: Union{Nothing, AbstractLocalOperator}
    Leave    :: Union{Nothing, InteractionTreeLeave}
    parent   :: Union{Nothing, InteractionTreeNode}
    children :: Vector{InteractionTreeNode}
end
```

支持 `AbstractTrees` 接口，可用 `print_tree` 可视化整棵树。

#### 添加相互作用

```julia
addIntr1!(tree, op, site; strength)          # 单体项
addIntr2!(tree, (op1,op2), (s1,s2); ...)     # 两体项
addIntr3!(tree, ops, sites; ...)             # 三体项
addIntr4!(tree, ops, sites; ...)             # 四体项
addIntr6!(tree, ops, sites; ...)             # 六体项
addIntrStr!(tree, ops, sites; ...)           # 弦算符（含 JW 弦）
```

### 3.9 局域空间 (LocalSpace)

提供常见量子系统的预构建算符，均以 Julia `module` 形式封装。

#### 自旋算符 (`Spin.jl`)

| 模块 | 物理空间 | 主要算符 |
|------|----------|----------|
| `SU₂Spin` | `Rep[SU₂](1/2 => 1)` | `SS`（S·S），`S2`（S²） |
| `U₁Spin` | `Rep[U₁](±1/2 => 1)` | `Sz`，`SzSz`，`S₊S₋`，`S₋S₊`，`S2` |
| `U₁Spin1` | `Rep[U₁](1,0,-1 => 1)` | 自旋-1 全套算符 |
| `SU₂Spin1` | `Rep[SU₂](1 => 1)` | `SS`，`S2` |

#### 费米子算符 (`Fermion.jl`)

| 模块 | 物理空间 | 主要算符 |
|------|----------|----------|
| `U₁SU₂Fermion` | `Rep[U₁×SU₂]((-1,0)=>1, (0,1/2)=>1, (1,0)=>1)` | `Z`（JW），`n`，`nd`，`F⁺F`，`FF⁺`，`SS` |
| `U₁U₁Fermion` | `Rep[U₁×U₁]((-1,0)=>1, (0,±1/2)=>1, (1,0)=>1)` | `n₊`，`n₋`，`Sz`，`nd`，自旋分辨费米算符，`S₊S₋` |
| `U₁Fermion` | `Rep[U₁](-1/2 => 1, 1/2 => 1)` | `Z`，`n`，`F⁺F`，`FF⁺` |

## 4. 核心算法

### 4.1 DMRG（密度矩阵重整化群）

**文件：** `src/Algorithm/DMRG.jl`

DMRG 是求量子多体基态的主算法，通过交替从左到右和从右到左扫描，逐格点优化 MPS 张量。

#### 公开接口

```julia
# 单格点 DMRG
DMRG1!(ψ::DenseMPS, H::SparseMPO;
    trunc   = ...,       # SVD 截断方案
    solver  = ...,       # Krylov 求解器
    N       = 4,         # 扫描次数
    Etol    = 1e-8,      # 能量收敛阈值
    Stol    = 1e-5,      # 熵收敛阈值
    λ       = 1.2,       # CBE 参数
    Nfull   = 4,         # 前几轮用 fullSVD
    subalgo = ...,       # 子算法
    GCsweep = 2,         # 每 N 次扫描做 GC
    GCsite  = -1         # 每 N 格点做 GC
)

# 双格点 DMRG
DMRG2!(ψ::DenseMPS, H::SparseMPO; kwargs...)
```

#### 单格点扫描流程（L2R 方向）

```
for site = 1:L-1
    1. [可选] CBE! 扩展键维度
    2. H_eff = proj1(env, site)          # 提取有效哈密顿量
    3. ψ_site = Krylov_eigsolve(H_eff)   # Lanczos 求基态
    4. (A, C, B) = SVD(ψ_site, trunc)    # SVD 截断
    5. env.layer[1][site] = A            # 左正交张量
    6. pushright!(env, site)             # 更新右环境
    7. 记录能量、熵
end
```

#### 收敛判据

```
|ΔE| < Etol  AND  |ΔS| < Stol
```

### 4.2 TDVP（时间依赖变分原理）

**文件：** `src/Algorithm/TDVP.jl`

TDVP 在 MPS 流形上求解薛定谔方程，实现实时和虚时演化。

#### 公开接口

```julia
# 单格点时间演化
TDVP1!(Env::Environment{3}, lst::AbstractVector; kwargs...)

# 双格点时间演化
TDVP2!(Env::Environment{3}, lst::AbstractVector; kwargs...)

# 包装函数（指定总时间和步数）
TDVP!(ψ::DenseMPS, H::SparseMPO, t::Number, Nt::Number; kwargs...)
```

#### 热化算法（tanTRG）

```julia
tanTRG1!(ρ::DenseMPO, H::SparseMPO, lsβ::Vector; kwargs...)  # 单格点
tanTRG2!(ρ::DenseMPO, H::SparseMPO, lsβ::Vector; kwargs...)  # 双格点
# 返回: (lsobj, lsinfo, lsF, lsE)
# lsF = 自由能列表, lsE = 能量列表
```

#### 时间步处理

每步时间为 `τ = (lst[i] - lst[i-1]) / 2`，一次完整演化包含 L2R + R2L 两个半步。

### 4.3 CBE（关联基展开）

**文件：** `src/Algorithm/CBE.jl`, `CBE2.jl`, `CBE3.jl`, `CBE-SVD.jl`

CBE 是键维度自适应扩展算法，用于在 DMRG/TDVP 中动态增加 MPS 键维度。

#### 调度逻辑（`dynamicSVD`）

```julia
if min(site, L - site) ≤ N
    # 边界格点：使用 fullSVD（精确但开销大）
    CBE!(env, CBEalgo{fullSVD}, info)
else
    # 内部格点：使用 randSVD（快速随机化）
    CBE!(env, CBEalgo{randSVD}, info)
end
```

#### 三种 SVD 方案对比

| 方案 | 适用场景 | 特点 |
|------|----------|------|
| `fullSVD` | 边界格点，小系统 | 精确，开销 O(D³) |
| `randSVD` | 内部格点，大系统 | 快速，参数 λ 控制采样数 |
| `dynamicSVD` | 通用（默认） | 自动切换，N 控制边界范围 |

SETTN 通过有限温度级数展开方法计算热力学量，适用于有限温度模拟。

## 6. 算法流程图

### DMRG 完整流程

```
输入: ψ (初始 MPS), H (SparseMPO)
         │
         ▼
构建 Environment([ψ, H, adjoint(ψ)])
         │
         ▼
初始化左/右环境（推进到初始格点）
         │
         ▼
┌────────────────────────────────────┐
│           扫描循环（N 次）          │
│                                    │
│  ┌─── L2R 扫描（site = 1 → L-1）  │
│  │    ┌─────────────────────────┐  │
│  │    │ [可选] CBE! 扩展键维度  │  │
│  │    │ H_eff = proj1(env,site) │  │
│  │    │ ψ' = Lanczos(H_eff, ψ) │  │
│  │    │ (A,C,B) = SVD(ψ')      │  │
│  │    │ 更新 ψ[site] = A        │  │
│  │    │ pushright!(env, site)   │  │
│  │    └─────────────────────────┘  │
│  │                                 │
│  └─── R2L 扫描（site = L → 2）   │
│       ┌─────────────────────────┐  │
│       │ [可选] CBE! 扩展键维度  │  │
│       │ H_eff = proj1(env,site) │  │
│       │ ψ' = Lanczos(H_eff, ψ) │  │
│       │ (A,C,B) = SVD(ψ')      │  │
│       │ 更新 ψ[site] = B        │  │
│       │ pushleft!(env, site)    │  │
│       └─────────────────────────┘  │
│                                    │
│  判断收敛: |ΔE| < Etol && |ΔS| < Stol
└────────────────────────────────────┘
         │
         ▼
输出: 基态 ψ, 基态能量 E₀
```

### 环境缩并示意

```
格点 i 的有效哈密顿量 H_eff：

  ┌──────┐   ┌──────┐   ┌──────┐
  │  L   │   │ H[i] │   │  R   │
  │  e   │───│      │───│  e   │
  │  f   │   │      │   │  f   │
  │  t   │   └──────┘   │  t   │
  └──┬───┘       │      └──┬───┘
     │         物理指标     │
     └─────────┬───────────┘
               │
           ψ[site]
```

## 7. 关键设计模式

### 7.1 参数化类型分发

大量利用 Julia 的多重分发和参数化类型，在编译期特化算法路径：

```julia
# 编译器根据 Sch（SingleSite/DoubleSite）选择不同实现
function DMRG!(env, alg::DMRGalgo{SingleSite, Alg}, info::DMRGsweepinfo{L2R})
    ...
end

function DMRG!(env, alg::DMRGalgo{DoubleSite, Alg}, info::DMRGsweepinfo{L2R})
    ...
end
```

### 7.2 稀疏/稠密双轨制

- `SparseMPO`：内存高效，存储非零局域算符的矩阵
- `DenseMPO`：计算高效，直接张量缩并
- 两者可互转（`densify.jl`）

### 7.3 惰性环境更新

环境张量增量更新，而非每步重建：

```julia
pushright!(env, site)  # 只更新 site → site+1 的部分缩并
pushleft!(env, site)   # 只更新 site → site-1 的部分缩并
```

### 7.4 伴随类型自动化

所有主要类型都有对应的伴随类型，保持代码对称性：

```julia
adjoint(ψ::DenseMPS)   -> AdjointMPS      # ⟨ψ|
adjoint(H::DenseMPO)   -> AdjointMPO      # H†
adjoint(T::MPSTensor)  -> AdjointMPSTensor
```

### 7.5 深度对称性集成

通过 TensorKit 在张量运算层面自动处理量子数守恒：
- 所有张量索引带有量子数标签
- 分解（QR、SVD）自动保持对称性
- 支持阿贝尔（U(1)）和非阿贝尔（SU(2)）对称性

### 7.6 多线程安全

使用原子计数器和锁保护并行环境更新，适用于多核计算。

## 9. 默认参数

| 参数 | DMRG | TDVP |
|------|------|------|
| Krylov 维度（`krylovdim`） | 8 | 32 |
| 最大迭代次数（`maxiter`） | 1 | 1 |
| 收敛容差（`tol`） | 1e-6 | 1e-8 |
| 正交化方法（`orth`） | ModifiedGramSchmidt | ModifiedGramSchmidt |
| 提前终止（`eager`） | true | true |
| 日志级别（`verbosity`） | 0（静默） | 0（静默） |

DMRG 使用较小的 Krylov 空间（8 维）和较低精度（1e-6），优先速度；TDVP 使用较大 Krylov 空间（32 维）和更高精度（1e-8），确保时间演化的数值稳定性。
